# v0.8.11

## Nouveautes

- Les scrappers peuvent maintenant lire les mangas dont chaque page est une page HTML avec une seule image.
- En mode `Pages`, un template comme `{{page}}` avec un selecteur d'image ouvre maintenant les URLs une par une, par exemple `/1/`, `/2/`, `/3/`, puis extrait l'image de chaque page.
- La validation de la configuration `Pages` teste aussi les premieres pages HTML sequentielles quand un template utilise `{{page}}` ou `{{pageIndex}}` avec un selecteur d'image.
- Les images des scrappers gerent mieux les URLs de fallback. Quand une source fournit plusieurs URLs possibles pour une couverture ou une miniature, l'application essaie les candidates et garde la premiere image exploitable.

## Corrections

- Les mangas lus depuis un template de pages HTML ne restent plus bloques sur la premiere page quand le lecteur avait deja sauvegarde une ancienne progression avec une seule page detectee.
- Le nombre de vignettes deja affichees sur une fiche ne limite plus la resolution des pages reelles du lecteur.
- Les scrappers peuvent recuperer le nombre de pages depuis la fiche, ce qui aide le lecteur et les telechargements a connaitre la taille attendue du manga.
- Les URLs d'images extraites par les scrappers sont mieux nettoyees quand elles contiennent des caracteres echappes ou des entites HTML.

## A savoir

- Pour profiter du nouveau comportement sur une fiche deja ouverte, recharge la fiche du scrapper avant d'ouvrir le lecteur.
- Les anciennes progressions de lecture restent conservees. Elles ne devraient plus empecher la detection de toutes les pages dans ce mode.
