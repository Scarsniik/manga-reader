# v0.11.0

## Nouveautes

- Un assistant visuel facilite maintenant la création des sélecteurs CSS dans la configuration des scrappers.
- Il affiche la page à analyser dans une fenêtre dédiée et permet de naviguer sans quitter la configuration.
- Des exemples positifs et négatifs peuvent être sélectionnés directement dans la page pour générer un sélecteur adapté.
- Le sélecteur obtenu peut être testé sur le HTML du scrapper et sur la page interactive avant d'être envoyé au formulaire.
- L'assistant permet également de modifier le pattern d'URL utilisé par le module.
- Une page de tag favori permet maintenant de rechercher des tags similaires sur tous les scrappers compatibles.
- Les résultats indiquent le nom du tag et son scrapper, classés selon leur proximité avec le favori ouvert sans laisser les termes trop génériques noyer la liste.
- La modale permet de filtrer les résultats par nom de tag ou de scrapper.
- Une étoile permet d'ajouter directement une source au favori ou de l'en retirer. Elle apparaît pleine et jaune quand la source est déjà associée.

## Corrections

- Le scraping d'une liste de tags conserve maintenant les pages déjà chargées lorsque la page suivante répond avec une erreur 404.
- Les noms et les liens des tags restent maintenant associés lorsque plusieurs tags partagent le même nom.

## A savoir

- Les redirections vers un autre domaine demandent une confirmation dans l'assistant visuel.
- Les sélecteurs qui présentent encore des erreurs peuvent être envoyés au formulaire après une confirmation explicite.
- La recherche utilise les listes de tags déjà enregistrées. Un scrapper sans liste disponible doit d'abord alimenter son cache de tags.
